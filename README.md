Fullstack ToDo Application (Nuxt 3 + Node.js)
Проект представляет собой менеджер задач с системой авторизации, реализованный на стеке Nuxt 3 (Frontend) и Express.js (Backend). Приложение поддерживает полноценный цикл работы с задачами и сохранение сессии пользователя.
Основные возможности
Авторизация пользователя на основе JWT-токенов.
Сохранение состояния сессии в Pinia Store с синхронизацией через localStorage.
Клиентский Middleware для защиты приватных роутов.
CRUD операции для управления списком задач (создание, чтение, удаление).
Динамическая генерация аватаров на основе email пользователя через внешний API.
Адаптивный интерфейс в стиле Glassmorphism.
Стек технологий
Frontend: Nuxt 3, Pinia (State Management), Vite, Vue 3.
Backend: Node.js, Express.js.
API: REST API, JWT Authentication.
Инструкция по развертыванию
Подготовка окружения
Убедитесь, что на вашем компьютере установлена Node.js версии 18.x или выше и менеджер пакетов npm.
Запуск Backend части
Перейдите в директорию серверной части командой **cd backend**
Установите необходимые зависимости командой **npm install**
Запустите сервер командой **npm start**
По умолчанию сервер будет доступен по адресу: http://localhost:5000
Запуск Frontend части
Откройте новое окно терминала и перейдите в директорию фронтенда командой cd frontend
Установите зависимости проекта командой **npm install**
Запустите приложение в режиме разработки командой **npm run dev**
Приложение будет доступно по адресу: http://localhost:3000
Архитектурные особенности
Безопасность и сессии
Для управления состоянием авторизации используется хранилище Pinia. При инициализации приложения Middleware проверяет наличие валидного токена в локальном хранилище браузера (localStorage) и восстанавливает сессию. Это исключает необходимость повторного входа при обновлении страницы (F5).
Взаимодействие с данными
Фронтенд взаимодействует с бэкендом через кастомный слой API. Все запросы к защищенным эндпоинтам автоматически включают заголовок Authorization: Bearer token, что обеспечивается централизованной логикой обработки сетевых вызовов.
Обработка ошибок
В приложении реализована базовая обработка ошибок авторизации и сетевых запросов. При истечении срока действия токена или попытке несанкционированного доступа пользователь перенаправляется на страницу логина.
