# Fullstack ToDo Application (Nuxt 3 + Node.js)

Современный менеджер задач с полноценной системой авторизации, построенный на разделении клиентской (Frontend) и серверной (Backend) частей. Приложение поддерживает полный цикл работы с задачами и сохранение состояния сессии.

---

## 🚀 Основные возможности

* Авторизация: Безопасный вход и регистрация пользователей с использованием JWT-токенов.
* Persistence Store: Интеграция Pinia с localStorage для сохранения сессии пользователя после перезагрузки страницы (F5).
* Route Protection: Клиентский Middleware, ограничивающий доступ к списку задач для неавторизованных гостей.
* Task Management: Полноценный цикл CRUD (создание, просмотр, удаление) задач через защищенное API.
* User UI: Динамическая генерация аватаров профиля и современный адаптивный дизайн в стиле Glassmorphism.

---

## 🛠 Стек технологий

* Frontend: Nuxt 3 (Vue 3), Pinia (State Management), Vite.
* Backend: Node.js, Express.js.
* API: REST API, JSON Web Tokens (JWT) для безопасности.

---
## 📦 Инструкция по развертыванию

Для корректной работы проекта необходима Node.js версии 18.x или выше.

### 1. Запуск Backend (Серверная часть)
1. Перейдите в директорию сервера: cd backend
2. Установите зависимости: npm install
3. Запустите сервер: npm start
*Сервер по умолчанию будет доступен по адресу: http://localhost:5000*

### 2. Запуск Frontend (Клиентская часть)
1. Откройте новое окно терминала и перейдите в директорию клиента: cd frontend
2. Установите зависимости: npm install
3. Запустите приложение: npm run dev
*Приложение будет доступно по адресу: http://localhost:3000*

## 🏗 Архитектурные решения

* Session Recovery: Приложение автоматически проверяет наличие токена в локальном хранилище при инициализации, восстанавливая профиль пользователя без лишних редиректов на страницу входа.
* Global API Layer: Все запросы к защищенным эндпоинтам автоматически включают заголовок Authorization: Bearer <token>, что реализовано через централизованный слой обработки запросов.
* Clean Repository: В проекте настроен общий файл .gitignore, исключающий попадание системных папок (node_modules, .nuxt) в репозиторий.
